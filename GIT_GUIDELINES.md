# Git Commit Guidelines

- Keep frontend and backend changes in separate commits when possible.
- Use descriptive commit messages: `feat(api): add order create route`.
- Prefix commits: `feat`, `fix`, `chore`, `docs`, `ci`, `refactor`.
- Create feature branches per task: `feature/brand-menu`, `feature/razorpay`.
- Use PRs and code review before merge to `main`.

Example workflow:
1. git checkout -b feature/brand-menu
2. work, commit changes (backend/ files only)
3. push and open PR
